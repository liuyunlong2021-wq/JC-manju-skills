# Skill 设计经验

> 来源：JC-短剧-世界模型（主要）+ JC-剧本转分镜 + JC-角色设计提示词 + JC-Wiki-记忆压缩

## 叙事三角引擎（来自 JC-短剧-世界模型）
- 旧模式：角色单引擎 → 催化剂只触碰红线。新模式：角色 × 道具 × 场景 三体联动
- **道具引擎**：物理法则+使用代价+叙事功能（麦高芬/资格认证/道德试炼/时间炸弹/真相容器/关系信物）
- **场景引擎**：空间逻辑+情绪锚点（权力压缩/身份外化/规则密室/情绪陷阱/时间胶囊）
- 催化剂 A/B/C/D：A优先三角、B/C按需降级。引擎级别是 LLM 内部质量检查，不输出给用户
- **不建追踪系统**：LLM 每场重读 wiki 自然判断道具/场景可用性，不需要手工记账
- 关键洞察：道具不是「好看的东西」，场景不是「背景板」——它们和角色一样，是冲突生成器

## 原型挪移法（来自 JC-短剧-世界模型）
- 创作初期不让用户从零想象——直接从 LLM 训练数据中的世界经典库提取引擎
- **角色**：五维提取——内核矛盾 / 决策算法 / 关系化学式 / 悲剧触发器 / 当代转译锚点
- **场景**：四维提取——空间逻辑 / 情绪锚点 / 材质语言 / 尺度签名
- **道具**：三维提取——物理法则 / 叙事功能 / 视觉签名
- 支持融合模式 + 零知识模式（故事上下文当过滤器），输出标准档案落袋到 wiki/

## 节拍索引法（来自 JC-短剧-世界模型 + JC-Wiki-记忆压缩）
- 检索知识库**按概念标签组织，不按来源章节号**——LLM 不是人类，不需要按页码翻
- 三层渐进披露：CLAUDE.md（一行）→ 概念索引.md（标签→[[双链]]）→ 原子笔记.md（细节）
- 每个索引条目一句话摘要——LLM 扫一眼就知道是什么，不需要点进去
- 状态追踪：目录.md frontmatter 写「最后操作」，一个月后回来三秒恢复上下文
- 关键洞察：Zettelkasten 在 AI 时代的用法——不是给人类翻的，是给 LLM 瞬间检索用的

## 通用摄入协议（来自 JC-Wiki-记忆压缩）
- 任何参考资料（小说/论文/电影/文献/网页）→ 统一模式：判断类型→拆解协议→原子笔记→概念索引
- 每种协议不同，但索引格式统一
- defuddle 预处理：URL → clean Markdown → .raw/
- Obsidian 兼容：所有输出遵循 [[双链]]、frontmatter、嵌入规范

## 分类档案法（来自 JC-剧本转分镜）
- 不要给笼统风格标签，让 LLM 按分类维度从训练数据中提取结构化知识档案
- 模板：锁定专家/作品 → 按维度提取档案 → 用户输入匹配 → 对照出选项 → 执行
- 分类维度要选 LLM 训练语料中自然存在的（电影按戏型、小说按情节类型）
- 输出要引用具体参考

## 像素参考法（分类档案法的进化，来自 JC-角色设计提示词）
- 当文字回忆不够精准时，直接用 Pinterest + view_image 搜真实参考图
- 搜索词必加 character turnaround sheet / environment design / prop design
- click_element 被拦截时用 navigate_page(pinURL)
- 提取视觉DNA → 人物小传差异融合 → JSON输出

## AI 生图卷轴法（来自 JC-分镜转分镜大图）
- AI 生图不需要规整宫格——用流动卷轴：镜头大小按情绪权重变化、箭头串联叙事、颜色虚线标角色动线
- 高潮镜 1/3-1/2 画面、过渡镜 1/8-1/12 画面
- GPT Image 2 的 @ 语法引用角色/场景/道具设计图
- 长镜(>8s)分时标注阶段演变

## Skill 结构
- 三层渐进披露：元数据(50字) → SKILL.md(<200行) → references/(按需加载)
- 每个 reference 必须写「什么时候读」
- 命名用功能导向，不用项目代号
- 措辞只写正面行为，去掉防御性

## 流程设计
- 阶段 0.5 全剧风格确认（多角色/多场景/多道具 Skill 必须）
- Wiki 联动：读 wiki/角色/*.md → 写 .design.json → 更新 CLAUDE.md
- JSON 输出：去掉散文、加 onImageText/closeups/distinctiveFeatures 等下游字段

## 迭代流程
审计功能跑题 → 升级核心机制 → 拆分结构 → 清理措辞/命名/触发词
