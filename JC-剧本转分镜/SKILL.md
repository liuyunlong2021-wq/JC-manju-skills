# 导演式分镜 —— 导演工作本

> 输入剧本 → 输出导演工作本。不是分镜表，是导演在监视器前说的话。核心：锁定导演对每种戏型的节奏决策，而非视觉标签。

## 理念

锁导演 ≠ 锁「暖橘灯、慢推轨」。锁的是他**面对每种戏型的节奏决策**——爱情戏怎么切、武打戏碎几镜、对峙留多少沉默、独角戏跟不跟人。

流程：锁定导演 → 提取按戏型分类的**场景节奏档案**（一次生成全剧复用）→ 逐场判定戏型 → 对照档案出 A/B/C/D 镜数选项 → 写成导演面对面的阐述。

每个镜头的输出是自然语言——画面、意图、手法三合一，直接适配下游视频模型。

## 核心规则

1. A/B/C/D 选项推进，不替用户决定
2. 风格必须「导演 + 作品」
3. **锁定的是导演对各种戏型的节奏处理方法**——不是泛泛的视觉风格
4. 分镜必须是**导演工作本**语言——画面+意图+手法三合一，不写技术参数表
5. 代词每镜独立，跨镜必须重写角色全名

## 流程总览

```
剧本 → 解析场次(阶段0) → 媒介分类(0.5) → 锁定导演作品(阶段1)
                                                    ↓
                                    导演场景节奏档案(阶段1.5)
                                    （按戏型分类，一次生成全剧复用）
                                                    ↓
                                    逐场匹配戏型+镜数选项(阶段2)
                                    A最多镜/B平均/C最少/D自定义
                                    → 用户逐场选，收集齐后进阶段3
                                                    ↓
                                    导演工作本(阶段3)
                                    镜号 + 导演阐述段落
```

## 跨轮次记忆

若分多次对话完成，每轮开始时回顾前文已选结果（锁定风格、节奏档案、各场镜数选择），确认「继续 / 重来」。节奏档案生成后保留在对话上下文中，避免重复生成。

## 剧本输入格式

## 阶段 0：解析剧本

按标准格式解析集号、场号、时间、场景、人物、△动作、台词。无标准格式则按「场/景/△/空行」拆分。确认：**A 接受 / B 重拆 / C 直接进风格**。

## 阶段 0.5：媒介分类

- **A 动漫** → `references/anime-style-bank.md`
- **B 真人** → `references/director-style-bank.md`

## 阶段 1：锁定导演作品（A/B/C/D）

基于剧本的题材/情绪/节奏，推荐 **3 个「导演 + 作品」**（A/B/C）+ D 自定义。

每个推荐含：
- 导演 + 作品名
- 为什么适合（一句话，点明该导演处理哪类戏型最贴合本剧本）
- 画幅：9:16 或 16:9

3 个选项覆盖不同方向（情绪作者型 / 节奏商业型 / 写实纪录型）。

要求：具体导演+具体作品。三个选项覆盖不同方向（情绪作者型/节奏商业型/写实纪录型）。

选 A/B/C → 锁定，进阶段 1.5。选 D → 收敛到具体「导演+作品」才能继续。

## 阶段 1.5：导演场景节奏档案

**这是整个 Skill 的核心**。锁定导演后，读取 `references/rhythm-archive-template.md`，按其模板从训练数据中提取该导演面对不同戏型的节奏处理方法——按戏型分类的结构化档案。

档案生成后，阶段 2 逐场匹配时直接查表——不需要重新推理这个导演的风格，只需要判定当前场是什么戏型、处于什么阶段。

## 阶段 2：逐场戏型匹配（A/B/C/D）

对每一场：

1. **判定戏型**：这场是什么戏型？处于什么阶段？
   - 例：「这是一场对峙/试探戏，小红与闯入者处于互相试探阶段，类似于《花样年华》中苏丽珍第一次在楼道遇见周慕云——两个陌生人之间微妙的张力」
2. **对照档案**：从节奏档案中取出该戏型对应的镜头数范围
3. **出四个选项**：

- **A：最多镜（X镜）** — 推荐理由（基于档案中该戏型的手法——多角度切换/物件穿插/碎切节奏）
- **B：平均（Y镜）** — 推荐理由（标准处理方式，不碎不疏）
- **C：最少镜（Z镜）** — 推荐理由（克制处理，长镜+留白，类似档案中某场参考戏的极简做法）
- **D：自定义** — 你想要的镜头数 + 节奏偏好

**推荐路径**：所有场次一次性列出戏型判定+选项，用户逐场回复。

## 阶段 3：导演工作本

读取 `references/director-notes-format.md`，按其格式输出：镜号独立一行，导演阐述段落紧随其后。

核心要求：
- 阐述是自然语言，画面+意图+手法三合一
- 每镜须明确「参考了该导演哪部作品的哪场戏」
- 代词每镜独立，跨镜必须重写角色全名
- 短镜（≤6秒）一段写完，长镜（>6秒）用时间分段

审阅输出格式见 `references/director-notes-format.md`。

## 输出文件

全部场次完成后，汇总输出 `analysis/shot-design.md`，格式与审阅输出一致。

## 约束

1. 风格「导演 + 作品」
2. 锁定的是节奏处理方法，不是视觉标签
3. 导演阐述必须融入「参考了该导演哪部作品的哪场戏」
4. 代词每镜独立
5. A/B/C/D 选项推进

## 风格参考库

- **真人** `references/director-style-bank.md`
- **动漫** `references/anime-style-bank.md`
- **节奏档案模板** `references/rhythm-archive-template.md` — 阶段 1.5 时读取
- **导演工作本格式** `references/director-notes-format.md` — 阶段 3 时读取

库只作格式锚点——提醒模型每个导演作品有哪些可参考的维度。真正有价值的是 LLM 从训练数据中对这些导演的深度知识。库内没有的作品按真实特征生成。
